# QQPGERP Agent 调用示例

默认地址按本地开发环境示例：

- Agent: `http://localhost:3101`
- ERP: `http://localhost:5801`

如果你的 `AGENT/.env` 里 `PORT` 不是 `3101`，把下面示例里的端口替换掉即可。

## 1. 健康检查

```http
GET http://localhost:3101/health
```

返回示例：

```json
{
  "status": "ok",
  "timestamp": "2026-03-13T07:00:00.000Z"
}
```

## 2. 自然语言调用

适合让 LLM 通过工具自动判断。


## 2.1 对话短期记忆

只要同一个 `sessionId` 连续调用，agent 就会保留最近几轮成功问答。

第二轮示例：

```http
POST http://localhost:3101/api/agent/chat
Content-Type: application/json

{
  "message": "再查一下刚才那个 SKU 的库存",
  "sessionId": "demo-session-001"
}
```

查看记忆：

```http
GET http://localhost:3101/api/agent/memory/demo-session-001
```

清空记忆：

```http
DELETE http://localhost:3101/api/agent/memory/demo-session-001
```

## 3. 固定履约流调用

适合业务系统直接调用，走确定性流程：

`仓库发货 -> 库存不足转采购 -> 采购协同供应商`

```http
POST http://localhost:3101/api/agent/fulfillment
Content-Type: application/json

{
  "items": [
    {
      "skuCode": "SKU-09871",
      "requiredQty": 200
    }
  ],
  "autoNotifySupplier": true
}
```

## 3.1 查看当前已注册 Skills


## 3.2 直接按 Skill 名执行

```http
POST http://localhost:3101/api/agent/skills/fulfillment_flow/execute
Content-Type: application/json

{
  "items": [
    {
      "skuCode": "SKU-09871",
      "requiredQty": 2
    }
  ],
  "autoNotifySupplier": false
}
```

字段说明：

- `items`: 必填，履约项数组
- `skuCode`: 必填，SKU 编码
- `requiredQty`: 必填，需要发货的数量，必须大于 0
- `autoNotifySupplier`: 可选，默认 `true`，库存不足时是否继续协同供应商

## 4. 库存预警调用

## 6. cURL 示例

### 6.1 自然语言

```bash
curl -X POST "http://localhost:3101/api/agent/chat" \
  -H "Content-Type: application/json" \
  -d "{\"message\":\"SKU-09871 先叫仓库发货，不够的话再去催采购，采购再找供应商\"}"
```

### 6.2 固定履约流

```bash
curl -X POST "http://localhost:3101/api/agent/fulfillment" \
  -H "Content-Type: application/json" \
  -d "{\"items\":[{\"skuCode\":\"SKU-09871\",\"requiredQty\":200}],\"autoNotifySupplier\":true}"
```

## 7. PowerShell 示例

### 7.1 自然语言

```powershell
$body = @{
  message = "SKU-09871 先叫仓库发货，不够的话再去催采购，采购再找供应商"
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Method Post -Uri "http://localhost:3101/api/agent/chat" `
  -ContentType "application/json" -Body $body
```

### 7.2 固定履约流

```powershell
$body = @{
  items = @(
    @{
      skuCode = "SKU-09871"
      requiredQty = 200
    }
  )
  autoNotifySupplier = $true
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Method Post -Uri "http://localhost:3101/api/agent/fulfillment" `
  -ContentType "application/json" -Body $body
```

## 8. 前端 fetch 示例

```javascript
const res = await fetch('http://localhost:3101/api/agent/fulfillment', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    items: [{ skuCode: 'SKU-09871', requiredQty: 200 }],
    autoNotifySupplier: true,
  }),
});

const data = await res.json();
console.log(data);
```

## 9. 注意事项

- 调用 Agent 接口不需要额外传 ERP token。
- Agent 内部会用 `ERP_API_TOKEN` 去调用 ERP。
- 当前已验证库存查询是通的。
- 当前采购申请自动建单会遇到 ERP `403` 权限限制，所以会回退成“待采购人工跟进”。
- 当前供应商通知接口会遇到 ERP `404`，所以会回退成“供应商需人工催办”。
